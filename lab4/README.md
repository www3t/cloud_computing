\### Шаг 2. Создание бакетов



\*\*Чем отличаются два способа управления доступом к бакетам в S3?\*\*



В Amazon S3 существуют два подхода к управлению доступом:



\*\*ACLs enabled (Access Control Lists)\*\* — классический метод. Позволяет управлять доступом на уровне каждого отдельного объекта. Можно сделать один файл публичным, другой — приватным. Удобен для экспериментов, но сложнее в управлении при большом количестве объектов.



\*\*Object Ownership Enforced (Bucket Owner Enforced)\*\* — современный метод. ACL полностью отключены, весь контроль доступа осуществляется через IAM-политики и Bucket Policy. Более безопасен и рекомендован AWS для production. Политики применяются ко всем объектам бакета централизованно.



\#### Публичный бакет `cc-lab4-pub-k21`



| Параметр | Значение |

|---|---|

| Имя | `cc-lab4-pub-k21` |

| Region | `eu-central-1` |

| Object Ownership | ACLs enabled |

| Block all public access | Отключено ✅ |



Нажата кнопка \*\*Create bucket\*\*.



\*\*Что означает опция "Block all public access" и зачем нужна данная настройка?\*\*



"Block all public access" — это защитная настройка, которая запрещает любой публичный доступ к бакету и его объектам, независимо от других политик. Она включает четыре подпункта:

\- Блокировка публичных ACL при загрузке объектов

\- Блокировка существующих публичных ACL

\- Блокировка публичных Bucket Policy

\- Блокировка существующих публичных Bucket Policy



Настройка нужна для защиты от случайной утечки данных. По умолчанию включена для всех новых бакетов. Для публичного хостинга (аватары, статический сайт) её нужно отключить, осознанно подтвердив это действие.

Загружен файл `user1.jpg`.  

В разделе \*\*Permissions\*\* выбрано \*\*Grant public-read access\*\*.  

Нажата кнопка \*\*Upload\*\*.



> 📸 \*\[скриншот: файл user1.jpg загружен в avatars/ с public-read доступом]\*



\*\*Чем отличается ключ (object key) от имени файла?\*\*



\*\*Имя файла\*\* — это просто название файла на локальном компьютере, например `user1.jpg`.



\*\*Object key (ключ объекта)\*\* — это полный уникальный идентификатор объекта внутри бакета S3, включающий весь "путь", например `avatars/user1.jpg`. В S3 нет настоящих папок — всё хранится в плоской структуре, а "папки" (`avatars/`) являются лишь префиксом ключа. Таким образом, `avatars/user1.jpg` — это один неделимый ключ, а не папка + файл.

\### Шаг 4. Загрузка объектов через AWS CLI



```bash

\# Загрузка user2.jpg в публичный бакет с публичным доступом

aws s3 cp s3-lab/public/avatars/user2.jpg \\

&#x20; s3://cc-lab4-pub-k21/avatars/user2.jpg \\

&#x20; --acl public-read



\# Загрузка logo.png в директорию content/

aws s3 cp s3-lab/public/content/logo.png \\

&#x20; s3://cc-lab4-pub-k21/content/logo.png \\

&#x20; --acl public-read



\# Загрузка activity.csv в приватный бакет (без публичного доступа)

aws s3 cp s3-lab/private/logs/activity.csv \\

&#x20; s3://cc-lab4-priv-k21/logs/activity.csv

```



> 📸 \*\[скриншот: вывод команд aws s3 cp — upload успешен]\*



\*\*В чём разница между командами `aws s3 cp`, `mv` и `sync`? Для чего используется флаг `--acl public-read`?\*\*



| Команда | Описание |

|---|---|

| `aws s3 cp` | Копирует объект из источника в назначение. Оригинал остаётся на месте. Аналог `cp` в Linux. |

| `aws s3 mv` | Перемещает объект — копирует в новое место и удаляет из старого. Аналог `mv` в Linux. |

| `aws s3 sync` | Синхронизирует директорию — копирует только новые или изменённые файлы. Идеально для регулярного обновления бакета из локальной папки. |



Флаг `--acl public-read` устанавливает ACL (Access Control List) объекта так, чтобы любой пользователь интернета мог читать (скачивать) этот объект по его URL без аутентификации.

\### Шаг 6. Версионирование объектов



Для обоих бакетов открыта вкладка \*\*Properties → Bucket Versioning → Edit → Enable → Save changes\*\*.



> 📸 \*\[скриншот: версионирование включено для cc-lab4-pub-k21]\*



Изменён файл `logo.png` и загружен повторно:



```bash

&#x20; s3://cc-lab4-pub-k21/content/logo.png \\

&#x20; --acl public-read

```



На вкладке \*\*Objects → Show versions\*\* видны две версии файла `logo.png` с разными Version ID.



> 📸 \*\[скриншот: две версии logo.png в бакете с разными Version ID]\*



\*\*Что произойдёт, если выключить версионирование после его включения?\*\*



Версионирование нельзя полностью отключить — можно только \*\*приостановить (Suspended)\*\*. При этом:

\- Все существующие версии объектов сохраняются и остаются доступны

\- Новые загрузки объектов получают Version ID = `null` (перезаписывают предыдущую null-версию)

\- Удаление объекта без указания Version ID создаёт Delete Marker, не удаляя реальные данные

\- Вернуть состояние "версионирование никогда не было включено" невозможно

\### Шаг 7. Lifecycle-правила для приватного бакета



Открыт бакет `cc-lab4-priv-k21` → вкладка \*\*Management → Lifecycle rules → Create rule\*\*.



\*\*Параметры правила:\*\*



| Параметр | Значение |

|---|---|

| Rule name | `logs-archive` |

| Prefix | `logs/` |



\*\*Actions (переходы между классами хранения):\*\*



| Действие | Срок |

|---|---|

| Transition to Standard-IA | Через 30 дней |

| Transition to Glacier Deep Archive | Через 365 дней |

| Expiration (удаление) | Через 1825 дней (5 лет) |



Нажата кнопка \*\*Create rule\*\*.



> 📸 \*\[скриншот: Lifecycle-правило logs-archive создано]\*



\*\*Что такое Storage Class в Amazon S3 и зачем они нужны?\*\*



Storage Class (класс хранения) — это характеристика объекта в S3, определяющая баланс между стоимостью хранения, доступностью и задержкой доступа к данным:



| Класс | Применение | Стоимость |

|---|---|---|

| \*\*S3 Standard\*\* | Часто используемые данные | Высокая |

| \*\*S3 Standard-IA\*\* | Редко используемые, нужна быстрая доступность | Ниже |

| \*\*S3 Glacier Instant\*\* | Архив с доступом за миллисекунды | Низкая |

| \*\*S3 Glacier Deep Archive\*\* | Долгосрочный архив, доступ за часы | Минимальная |



Lifecycle-правила позволяют автоматически переводить объекты между классами по расписанию, оптимизируя затраты на хранение без ручного вмешательства.

\### Шаг 9. Дополнительное задание — PHP-приложение с AWS SDK



Реализовано PHP-приложение для загрузки файлов в S3 с хранением метаданных в SQLite.



\*\*Функциональность:\*\*

\- Форма загрузки изображений

\- Генерация уникального имени файла (timestamp + случайный hex)

\- Загрузка в S3 бакет с ACL `public-read`

\- Сохранение метаданных в SQLite: оригинальное имя, уникальное имя, URL, дата загрузки

\- Страница со списком всех загруженных файлов с превью



\*\*Установка зависимостей:\*\*



```bash

composer require aws/aws-sdk-php

```



\*\*Ключевые части кода:\*\*



```php

// Генерация уникального имени файла

$uniqueName = date('Ymd\_His') . '\_' . bin2hex(random\_bytes(4)) . '.' . $ext;

$s3Key = 'avatars/' . $uniqueName;



// Загрузка в S3

$result = $s3->putObject(\[

&#x20;   'Bucket'     => BUCKET\_NAME,

&#x20;   'Key'        => $s3Key,

&#x20;   'SourceFile' => $file\['tmp\_name'],

&#x20;   'ACL'        => 'public-read',

]);



// Сохранение в SQLite

$stmt = $db->prepare("INSERT INTO uploads (original\_name, unique\_name, url) VALUES (?, ?, ?)");

$stmt->execute(\[$originalName, $uniqueName, $result\['ObjectURL']]);

```

\## Ответы на контрольные вопросы



| № | Вопрос | Ответ |

|---|---|---|

| 1 | Чем отличаются ACL и Object Ownership Enforced? | ACL — управление доступом на уровне каждого объекта; Enforced — централизованный контроль через IAM/Bucket Policy, ACL отключены |

| 2 | Что означает "Block all public access"? | Защита от случайной публикации данных. Блокирует все публичные ACL и политики. По умолчанию включена для всех бакетов |

| 3 | Чем отличается object key от имени файла? | Имя файла — локальное название. Object key — полный уникальный путь в бакете, включая все префиксы (напр. `avatars/user1.jpg`) |

| 4 | Разница между cp, mv и sync? | cp — копирует; mv — перемещает (копирует + удаляет); sync — синхронизирует директорию, копируя только изменения |

| 5 | Что такое --acl public-read? | Флаг ACL, разрешающий любому пользователю интернета читать объект по его URL без аутентификации |

| 6 | Что будет при выключении версионирования? | Нельзя полностью отключить — только Suspended. Старые версии сохраняются, новые загрузки получают Version ID = null |

| 7 | Что такое Storage Class? | Характеристика объекта, определяющая стоимость и скорость доступа. От Standard (дорого, быстро) до Glacier Deep Archive (дёшево, медленно) |
